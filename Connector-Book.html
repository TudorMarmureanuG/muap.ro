Connector.exe Support




Connect your Multiapp application to certain services that ensure its proper and complete functionality. It needs to be placed in the current folder where Multiapp.exe is located and should be in the subfolder named muap.ro (ex: Multiapp Enterprise/muap.ro). This library cannot be run individually; it runs together with Multiapp.exe.



This application comes pre-installed in the Setup and automatically runs Muap services when Multiapp.exe is launched. If it is deleted from the system, Multiapp.exe will not function optimally and correctly.




Video caption: Let's assume a user wants to create another Multiapp account. Everything proceeds normally: I complete the account setup, provide the user with the username, password, and the additional necessary details. Now, this user wants to go to a café with their laptop but doesn't have internet access there. However, they still want to use that account. Unfortunately, access to that account doesn't work offline. The main issue is that Multiapp uses a service called Connector, which runs alongside the application and needs to communicate with our server to function properly. Without an internet connection, Connector cannot perform this communication, meaning the user cannot access the account. Each time the user tries to log into the Multiapp account, the application must sync data with the server because the account data is not stored locally on the user's laptop. Even if the user has previously connected to the internet and successfully accessed the account, once the internet connection is lost, the account is no longer functional offline. To solve this problem and prevent such situations in the future, I will create a new installer (wizard) for the Multiapp application whenever a new account needs to be created. After reinstalling Multiapp on the user's device, the account can function without an internet connection. This way, the user will be able to use the Multiapp account offline, wherever they are, even when they do not have access to the internet.
